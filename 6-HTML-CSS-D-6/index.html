Explanation of "flex-direction"

So, first we gave a style to container class div, (display:flex;). This basically converts the parent element ("container" div) to a flex-container.

And this converts all the elements inside the flex-container to flex-items. Simple as that.

NOTE: The flex-items are the ones who are immediate children to the flex-container and must be an HTML element. Any text written inside the flex-container are not flex-items

Go down for more flexbox model knowledge.

You previously saw that as soon as you change the flex-direction:column;. The flex-items get aligned vertically.

By default, the direction of the flex-items is row or horizontal.

Cross & Main Axis and "justify-content"

Before flexbox model was introduced in Web Designing it was a pain to align children elements accordingly inside the parent element. Flexbox model has two awesome properties that helps us to do so very easily.

Before we move on to that we need to know what cross-axis and main-axis are.

These two axises are imaginary lines that passes through a flex-container. We only imagine them to make a reference. Let's see what are they.

Go down to know about the two axises

Cross-Axis and Main-Axis depend upon the property, flex-direction of the flex-container.

It has five values:

align-content

Similar to the justify-content is another flex-box property called align-content.

As justify-content works along the "main-axis", in a similar fashion,
align-content works along the "cross-axis"

Go down to see align-content values

align-content has similar values to that of justify-content with a extra value:

  • flex-start (default value)
  • flex-end
  • center
  • space-around
  • space-between
  • stretch

stretch basically stretches all the elements' height to fill the maximum available height of the parent element.

Flex-Item properties

We were till now solely focused on the flex-container CSS properties. There are two prominent CSS properties for flex-item which we will use a lot.

  • flex-grow / flex
  • align-self

The flex-grow determines the width/height of the particular flex-item. While align-self tells us where the particular element should be positioned.

Go down to see more about flex-grow

Flex-grow / flex

flex-grow determines the length of the flex-item along the main-axis.

Let's modify the style element in 3-flexbox.html file a little bit.


<style>
        body{
            margin:0;
        }
        .container{
            display: flex;
            flex-direction: row;
            justify-content: flex-start;
            min-height:50vh;
            /* align-items: center; */
            align-content: flex-end;
            width: 100vw;
            flex-wrap: wrap;
            background-color: #000;
        }
    
        .item {
            color: #fff;
            font-size: 3em;
            padding-left: 2em;
            padding-right: 2em;
            padding-top: 0.5em;
            padding-bottom: 0.5em;
        }
    
        #i1{
            background-color: #ff0000;
            flex-grow: 1;
        }
        #i2{background-color: #ff7c00;}
        #i3{
            background-color: #fff700;
            flex-grow: 2;
        }
        #i4{background-color: #64ff00;}
        #i5{background-color: #00ff8b;}
        #i6{background-color: #0070ff;}
        #i7{background-color: #0f00ff;}
        #i8{background-color: #d100ff;}
>/style>
Go down to see more

We made special changes to the style of elements with id: i1 & i3

Notice that the length of these two elements grow and the length of i3 is greater than i1.

NOTE: The default value of flex-grow is 0. These elements with flex-grow:0 will retain their original size. While the ones with a number greater than 0 will increase and take up rest of the space left in the flex-container.

Go down to see more on flex-grow

So, the calculation goes like this:
In our example the flex-items with flex-grow > 0 are i1 & i3.

So, i1 will take up 1 / 1+0+2+0+0+0+0+0 = 1/3 of the remaining space.
& i2 will take 2 / 1+2+0+0+.. = 2/3 of the remaining space along the main-axis.

That's about it. You will get hold of this once we apply it to a project.

Go down to see more about align-self.

align-self is similar to align-content but it only applies to a particular flex-item instead of all the flex-items.

You have to declare this style property inside the flex-item CSS. Its values are:

  • auto (default) -> basically has alignment declared in flex-container
  • flex-start -> aligns a flex-item to the start of the main-axis
  • flex-end -> aligns a flex-item to the end of the main-axis
  • center -> aligns a flex-item to the center of the main-axis
  • stretch -> stretches a flex-item to the container's full length along cross-axis
  • baseline

Explaining the code:

The code is pretty simple:

@media-query screen and (max-width: 424px){...

This line means that when the screen size has a width of 424px or below, activate the styles below between {...}. 


@media screen and (max-width: 424px) {
    .mobile{ color: red;  }
    .tablet { color: lightcyan;  }
    .desktop { color: lightcyan;  }
}

This means when screen has a size of 424px or below, change the style of .mobile class to color:red; and the styles of other two classes to color:lightcyan;
